## Overview

The Proofly API uses two authentication methods:

1. **API Key Authentication** - For programmatic access to API endpoints
2. **Bearer Token Authentication** - For user-specific operations like managing API keys

## API Key Authentication

Most API endpoints require an API key in the request header. API keys start with `sk_` and should be kept secret.

### Using API Keys

Include your API key in the `X-API-Key` header:

<CodeGroup>

```bash cURL
curl -X GET https://api.proofly.joinplank.com/api/jobs/job-id \
  -H "X-API-Key: sk_your_api_key_here"
```

```javascript JavaScript
const response = await fetch('https://api.proofly.joinplank.com/api/jobs/job-id', {
  headers: {
    'X-API-Key': 'sk_your_api_key_here'
  }
});
```

```typescript TypeScript
const response = await fetch('https://api.proofly.joinplank.com/api/jobs/job-id', {
  headers: {
    'X-API-Key': 'sk_your_api_key_here'
  }
});
```

</CodeGroup>

## User Login

To create and manage API keys, you first need to authenticate as a user and get a bearer token.

<Endpoint method="POST" url="/api/login" />

### Request Body

```json
{
  "email": "user@example.com",
  "password": "your-password"
}
```

### Response

```json
{
  "userId": "uuid",
  "email": "user@example.com",
  "accessToken": "jwt-token-here",
  "refreshToken": "refresh-token-here",
  "expiresAt": 1234567890
}
```

<CodeGroup>

```bash cURL
curl -X POST https://api.proofly.joinplank.com/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "your-password"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.proofly.joinplank.com/api/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'your-password'
  })
});

const { accessToken } = await response.json();
```

```typescript TypeScript
interface LoginResponse {
  userId: string;
  email: string;
  accessToken: string;
  refreshToken?: string;
  expiresAt?: number;
}

const response = await fetch('https://api.proofly.joinplank.com/api/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'your-password'
  })
});

const data = await response.json() as LoginResponse;
const { accessToken } = data;
```

</CodeGroup>

## Bearer Token Authentication

After logging in, use the `accessToken` as a Bearer token for user-specific endpoints that require user authentication.

<CodeGroup>

```bash cURL
curl -X GET https://api.proofly.joinplank.com/api/user-endpoint \
  -H "Authorization: Bearer your-access-token-here"
```

```javascript JavaScript
const response = await fetch('https://api.proofly.joinplank.com/api/user-endpoint', {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});
```

```typescript TypeScript
const response = await fetch('https://api.proofly.joinplank.com/api/user-endpoint', {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});
```

</CodeGroup>

## Getting Your API Key

API keys are managed through the frontend application. After logging in, you can create and manage API keys through the dashboard interface.

## Security Best Practices

<AccordionGroup>
  <Accordion title="Keep API Keys Secret">
    Never commit API keys to version control. Use environment variables or secure secret management systems.
  </Accordion>
  
  <Accordion title="Rotate Keys Regularly">
    Periodically rotate your API keys to minimize risk if a key is compromised.
  </Accordion>
  
  <Accordion title="Use Different Keys for Different Environments">
    Use separate API keys for development, staging, and production environments.
  </Accordion>
  
  <Accordion title="Revoke Compromised Keys Immediately">
    If you suspect an API key has been compromised, delete it immediately and create a new one.
  </Accordion>
  
  <Accordion title="Use HTTPS Only">
    Always make API requests over HTTPS to protect your credentials in transit.
  </Accordion>
</AccordionGroup>

## Error Responses

### 401 Unauthorized

Returned when:
- API key is missing or invalid
- Bearer token is missing, invalid, or expired

```json
{
  "error": "Unauthorized",
  "message": "Invalid API key"
}
```

### 403 Forbidden

Returned when:
- API key doesn't have permission for the requested resource
- Bearer token doesn't have required permissions

```json
{
  "error": "Forbidden",
  "message": "Insufficient permissions"
}
```

See the [Error Handling guide](/guides/error-handling) for more details on handling authentication errors.
